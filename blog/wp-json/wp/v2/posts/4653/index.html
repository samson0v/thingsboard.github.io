We are thrilled to announce the release of ThingsBoard version 3.7! Highlights include the migration to JDK 17, enriched localization support, database structure optimizations, enhancements to existing widgets, and the addition of several new ones.<\/p>\n\n\n\n

Performance and security<\/h4>\n\n\n\n

We’ve worked on a bunch of behind-the-scenes improvements to make the software more efficient and secure. Here’s a look at what’s changed:<\/p>\n\n\n\n

    \n
  1. Migration to the Java 17 LTS release:<\/strong> We’ve upgraded from JDK 11 to JDK 17, bringing in new features and improvements that help the software run more efficiently and securely.<\/li>\n\n\n\n
  2. Spring Framework Update:<\/strong> We’ve updated Spring from 5.3.26 to 6.1.8, providing improved performance, enhanced security, and compatibility with the latest Java 17 features.<\/li>\n\n\n\n
  3. Docker Image Update:<\/strong> Our Docker images have been updated from bullseye-slim to bookworm-slim. This update includes bug fixes and new features, ensuring a smoother and more secure deployment environment. See PR #11<\/a> for more details.<\/li>\n\n\n\n
  4. Housekeeping service:<\/strong> We’ve added a new service that takes care of long-running operations in the background. For example, it removes old alarms, attributes, and telemetry records for entities you delete from the platform. See PR #10201<\/a> for more details. <\/li>\n\n\n\n
  5. Database structure optimization:<\/strong> We have reduced the size of the table that stores attributes by ~30%. This optimization ensures the software runs more efficiently, even on devices with limited resources. See PR #9850<\/a> for more details.<\/li>\n<\/ol>\n\n\n\n

    Charts Library Upgrade<\/h4>\n\n\n\n

    We are upgrading our widgets to utilize Apache ECharts, introducing new capabilities and features for improved data visualization. This upgrade applies specifically to the following widgets: State chart, Pie chart, Radar chart, Polar area, and Bars. All these widgets now include a basic configuration form for a more intuitive setup experience.<\/p>\n\n\n\n
